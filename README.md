# RoboMunch — Artist Chatbot

EE 471 Modern Software Development Practices and Technologies — **Mini Project #3**.

A web-based artist chatbot inspired by Edvard Munch. Two studios:

- **Art Studio** — type a prompt, press the paint button, get an image generated by Stable Diffusion.
- **Chat Studio** — chat with RoboMunch (SmolLM2-360M-Instruct) in text, or use the microphone to dictate via Whisper speech-to-text.

The UI is a single-page vanilla HTML/CSS/JS app styled to match the project mockup. The backend is FastAPI; all models run locally (NVIDIA GPU recommended).

## Setup

Requires Python 3.10+ and (recommended) an NVIDIA GPU with CUDA.

```powershell
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
# CUDA wheel for torch (skip if you already have a working torch install):
# pip install torch --index-url https://download.pytorch.org/whl/cu121
```

The first run will download the models from the HuggingFace Hub:
- `runwayml/stable-diffusion-v1-5` (~4 GB)
- `HuggingFaceTB/SmolLM2-360M-Instruct` (~720 MB)
- `openai/whisper-base` (~150 MB)

## Run

```powershell
cd backend
python main.py
```

Then open <http://127.0.0.1:8000/> — FastAPI serves the frontend at the root.

## Endpoints

| Method | Path                  | Body                             | Returns                        |
|--------|-----------------------|----------------------------------|--------------------------------|
| POST   | `/api/generate-image` | `{ "prompt": "..." }`            | `{ "image": "data:image/png;base64,..." }` |
| POST   | `/api/chat`           | `{ "message": "...", "history": [...] }` | `{ "reply": "..." }` |
| POST   | `/api/transcribe`     | multipart `audio`                | `{ "text": "..." }`            |
| GET    | `/api/health`         | —                                | `{ "ok": true }`               |

## Demo flow (for the screen recording)
1. Type into the chat: *"Hello Edvard! Generate me a prompt to draw a digital art image. Keep it within 20 words."*
2. RoboMunch replies with a short vivid prompt.
3. Copy the reply into the **prompt input** at the top, press the **paint button**, watch the image render.
4. Send a follow-up chat message (e.g., "Make it more cosmic.") to reach 4 total chat messages.
5. Click the **microphone button**, speak a sentence, click again to stop — the transcription appears in the chat input.

## Notes
- First inference of each model is slow (model warmup). Subsequent calls are fast.
- If VRAM is tight, the image task already uses `float16` and `enable_attention_slicing()`.
- Browser compatibility: Chrome / Edge / Firefox. The microphone uses `MediaRecorder` (webm).
- No HuggingFace token is required — all models are public.
